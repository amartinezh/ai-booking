# ============================================================
# service-synonyms.txt
# Tesauro GENÉRICO de servicios clínicos (válido para cualquier clínica).
#
# Objetivo: cuando el paciente usa un sinónimo o lenguaje natural que NO
# comparte substring con el nombre del catálogo (ej. "rayos x" vs
# "Radiología e Imágenes Diagnósticas"), expandimos su frase a las
# "anclas" del concepto y las probamos contra el catálogo REAL de la
# clínica con el matcher determinista (sin LLM, anti-alucinación).
#
# NO mapea a un serviceId: cada clínica tiene los suyos. Mapea a CONCEPTOS
# genéricos; las anclas son las palabras que sí suelen aparecer en el
# nombre del catálogo. Si ninguna ancla coincide con el catálogo de esa
# clínica, el flujo cae al nivel LLM como antes.
#
# Formato:
#   - Líneas que empiezan con '#' o vacías → ignoradas.
#   - [concepto: <clave>]  → inicia un concepto.
#   - anclas:   a | b | c  → frases esperadas en el NOMBRE del catálogo.
#   - sinonimos: x | y | z → lenguaje natural del paciente (no comparte substring).
#   - Todo se normaliza (minúsculas, sin tildes) al cargar; escríbalo en
#     minúsculas y sin tildes por claridad. Evite tokens sueltos demasiado
#     genéricos ("consulta", "medico", "examenes") → producen falsos positivos.
#
# Para editar en caliente: ajuste el archivo y llame a reloadPatterns().
# ============================================================
